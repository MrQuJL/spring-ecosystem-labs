根据您的需求，我将为您设计并实现一个专门用于更新客户状态的接口。

### 1. 接口设计
遵循 RESTful 风格，建议使用 `PATCH` 方法，因为这是对资源的局部更新。

*   **URL**: `/api/v1/customer/{id}/status`
*   **Method**: `PATCH`
*   **描述**: 更新指定客户的状态（如冻结/解冻）。
*   **参数**:
    *   `id` (Path Variable): 客户ID
    *   `status` (Query Parameter): 新的状态码 (0: 冻结, 1: 正常)

### 2. 代码变更计划

#### 2.1 修改 Service 层
*   **接口 (`ICustomerService`)**: 新增 `boolean updateStatus(Long id, Integer status)` 方法定义。
*   **实现 (`CustomerServiceImpl`)**: 实现该方法。
    *   查询客户是否存在。
    *   校验状态码是否合法（使用 `CustomerStatusEnum`）。
    *   更新状态并保存。

#### 2.2 修改 Controller 层 (`CustomerController`)
*   新增 `updateStatus` 方法。
*   使用 `@PatchMapping("/{id}/status")`。
*   添加 Swagger 注解 `@Operation(summary = "更新客户状态")`。
*   调用 Service 层方法并返回结果。

### 3. 示例调用
```http
PATCH /api/v1/customer/1/status?status=0
```
